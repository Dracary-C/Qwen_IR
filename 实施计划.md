# Qwen 辅助图像修复三周实验计划

## 1\. 总体目标

在单张 A100 40GB、三周预算内，依次回答四个问题：

1. 当前恢复 backbone 本身是否可靠？
2. 正确退化先验是否确实能提升恢复？
3. Qwen prior 与 calibrated confidence 是否有效且抗误判？
4. layout 驱动的 depth/segmentation/DoG 空间专家是否优于全局 FiLM？
5. codex建议在S3以后再开始考虑调整loss等

主实验统一使用：

* Conditional UNet 恢复主干、256×256、Direct-GT、L1；
* batch size 12、20k steps、AdamW 1e-4、cosine、warmup 1k；
* 完整五任务 Train split，采用类别均衡采样；
* 完整 Val 2551 张作为正式评测；
* 固定 seed 1234 做消融，最终 baseline/proposed 使用 3 seeds；
* 主指标为五任务 macro PSNR/SSIM，避免 DehazeFormer 数量优势主导平均值。

## 2\. 必要接口与实验基础设施

* 将脆弱的 `prior\\\_vector\\\_21` 升级为命名式 `StructuredPriorV2`：

  * `severity\\\_5`
  * `main\\\_logits\\\_5`
  * `main\\\_probs\\\_5`
  * `layout\\\_10`
  * `raw\\\_margin`
  * `calibrated\\\_confidence`
  * 保留旧 21D vector 读取兼容；`serious/severe` 均归一化为数值 1.0。
* 从训练集 Qwen 原始平均 log-prob 拟合一个 temperature，输出校准后的五类分布；以 `max(calibrated\\\_probs)` 作为 confidence。验证集报告 NLL、ECE、Brier、错误检测 AUROC、AURC。
* 新增统一消融开关：

  * `none`
  * `zero\\\_prior`
  * `oracle\\\_type`
  * `qwen\\\_probs`
  * `qwen\\\_probs\\\_severity`
  * `confidence\\\_gate`
  * `global\\\_layout`
  * `spatial\\\_experts`
* A4-clean 的 confidence gate 固定为：
\[
z=c,z\_{\\text{Qwen}}+(1-c)z\_{\\text{unknown}}
]
其中 `z\\\_unknown` 为可学习通用条件。A4-clean 不使用 condition
dropout 或 shuffled-prior corruption，以单独验证 confidence gate。
dropout/corruption 仅作为后续独立鲁棒性消融，启用前需要重新确认。
* 三空间专家复用现有 Depth-Anything、SegFormer、DoG：

  * depth gate：`confidence × depth\\\_dependent`
  * segmentation gate：`confidence × max(local\\\_region, object\\\_specific)`
  * DoG gate：`confidence × max(texture\\\_dependent, directional, discrete)`
  * `uncertain=true` 时所有空间 gate 为 0。
* 专家输出多尺度空间 `gamma/beta` map，通过 SFT 调制 UNet feature；禁止再次对空间 token 求均值。训练阶段缓存 depth/seg/DoG map，在线推理阶段实时计算。
* 所有 run 保存 config、seed、checkpoint、每图指标、Qwen 输出、路由权重和运行时间，结果汇总脚本执行 macro average 与 paired bootstrap 95% CI。

## 3\. 三周实验流程

### 第 1 阶段：恢复能力与先验上界，Day 1–4

|ID|条件|训练|
|-|-|-|
|G0|当前 LOL-v2 单任务、无 prior|完成现有 20k|
|A0|五任务 plain UNet|20k|
|A0z|架构相同但 prior 全零|5k|
|A1|GT 主退化 one-hot、固定 severity=1|先 5k，通过后补到 20k|

闸门：

* G0 必须比输入 LQ 提升至少 1 dB，且 batch test 与训练评估无数值范围差异。
* A0z 与 A0 在同 step 差异应小于 0.1 dB。
* A1 在 5k 时应比 A0 提升至少 0.15 dB macro PSNR，且至少 3/5 任务不下降；否则停止后续 prior 实验，先修复注入路径。

### 第 2 阶段：Qwen prior 与 confidence，Day 5–10

|ID|条件|训练|
|-|-|-|
|A2|calibrated main probabilities|5k 筛选|
|A3|A2 + severity\_5|20k|
|A4-clean|A3 + confidence gate；无 dropout/corruption|10k（当前 sample 协议）|
|A5.0|A3 + global layout FiLM（不使用 confidence blend）|先 5k 筛选|
残差输出作为独立参数化消融，不与 prior/layout 变量混合：

|ID|对应 image 实验|唯一变化|
|-|-|-|
|R0|A0|`prediction_target: residual`|
|R1|A1|`prediction_target: residual`|
|R2|A2|`prediction_target: residual`|
|R3|A3|`prediction_target: residual`|

R 系列保持对应 A 实验的数据、prior、batch、seed、步数和调度器不变。若 R3 明显优于 A3，再决定是否让 A5.0 使用 residual backbone。

每个模型在验证时额外执行：

* 正确 prior、全零 prior、均匀 prior、batch-shuffled prior、强制错误 top-1；
* 按 confidence bottom 20% / middle 60% / top 20% 分桶；
* 按 Qwen 判断正确/错误分别统计恢复增益。

闸门：

* A3 相对 A0 至少提升 0.2 dB macro PSNR，且任何任务下降不超过 0.2 dB。
* A4 相对 A3：bottom-20% 至少提升 0.3 dB，高 confidence 子集下降不超过 0.05 dB。
* 若 A5.0 没有至少 0.1 dB 增益，则将其作为“全局 layout 无效”的证据，不再沿用其注入方式。

### 第 3 阶段：局部/复合控制集与空间路由，Day 8–15

构造 clean-identity 无交叉的控制集：

* 10k train / 1k val / 1k test；
* 50% 单退化、50% 两种复合退化；
* 五类与 mild/moderate/severe 均衡；
* 包含 global、smooth local region、object-specific、depth-dependent、shadow-dependent、texture/directional 布局；
* 保存每类真实 strength map、综合 mask、参数、退化顺序和 oracle layout；
* Qwen 对整个控制集生成结构化输出；训练可缓存，但正式在线评测禁止读取缓存。

在“完整原始数据 : 控制数据 = 1:1”的混合采样下从头训练：

|ID|条件|Seeds|
|-|-|-|
|S0|无 prior mixed baseline|1234、2026、3407|
|S1|confidence prior，无 layout|1234|
|S2|confidence + global layout|1234|
|S3|confidence + depth/seg/DoG 空间专家|1234、2026、3407|

空间实验同时评估：

* Qwen layout、oracle layout、随机 layout；
* test-time 分别关闭 depth、seg、DoG；
* 区域内 PSNR/SSIM、区域外 preservation PSNR；
* 单退化与复合退化分别统计；
* 路由图与 GT strength map 的相关性/IoU；
* 同一 LQ 下切换 layout 的反事实响应。

通过标准：

* S3 相对 S1：局部区域内至少 +0.3 dB；
* 复合退化整体至少 +0.2 dB；
* 区域外下降不超过 0.1 dB；
* 原始完整 Val macro PSNR 不得下降超过 0.05 dB；
* oracle layout 必须优于随机 layout，否则不能声称网络利用了 layout。

### 第 4 阶段：最终验证与在线系统，Day 16–21

* 完成 S0 与 S3 的 3-seed 均值、标准差和 paired bootstrap 95% CI。
* 在固定 100 张样本上测试真正在线流程：

  * Qwen preprocessing/generation/scoring；
  * calibration；
  * depth/seg/DoG；
  * restoration；
  * 总延迟、峰值显存、失败率。
* 分别报告 cold start 和模型已加载后的 warm batch-1 延迟；关闭 JSON cache。
* 优先让 Qwen 与 restoration 常驻同一卡，峰值显存目标 `<38 GiB`；若 OOM，采用 Qwen 完成后卸载再加载 restoration 的确定性顺序回退。
* 仅在 Direct-GT 全部过关后，对 S0/S3 各做 5k SDE fine-tune，并在 200 张分层样本上跑完整 100-step rollout；若再次出现明显伪影，则论文主结论限定为 Direct-GT，不包装 SDE 结果。

## 4\. 测试与最终产物

* 单元测试：旧 21D/V2 schema、`serious/severe`、confidence 极值、`uncertain`、专家 gate、空间尺寸和缓存一致性。
* 数据测试：固定 seed 可复现、clean identity 无泄漏、mask 范围正确、各类型/严重度/layout 分布均衡。
* 端到端测试：缓存 prior 与在线 Qwen 对同一图产生一致结构；batch 与 single-image 恢复结果一致。
* 最终形成四张主表：

  1. 原始五任务逐步消融；
  2. confidence 分桶与错误 prior 鲁棒性；
  3. 局部/复合退化及区域指标；
  4. 在线延迟、显存和专家消融。
* 最终保留的论文主模型为 S3；若空间闸门失败但 A4 通过，则收缩为“校准置信度与错误先验鲁棒性”论文主线。

## 5\. 固定假设

* 只使用当前 Conditional UNet，不在本轮引入 Restormer/NAFNet。
* Qwen3-VL-8B 是最终在线组件；训练期间允许缓存其输出。
* 主结论建立在 Direct-GT 配对指标上，真实图像与 SDE 仅作补充。
* 三周内优先保证 baseline、oracle、confidence 和 S3 完整；时间不足时，A2、A5.0、S1、S2 保留单 seed 或 5k 消融，不削减 S0/S3 的 3-seed 正式结果。
